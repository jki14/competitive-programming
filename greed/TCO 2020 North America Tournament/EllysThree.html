TCO 2020 North America Tournament 1 600 EllysThree

Problem Statement

Elly has a rectangular matrix with N rows and M columns. Each cell contains a digit (0-9). The girl defines a "path" from the cell in the upper left corner (row 0, column 0) to the lower right one (row N-1, column M-1) as a sequence of cells sharing a side going only down or to the right.

Each path has an associated value – that is the integer we get when we concatenate the digits in the visited cells, in the order we visit them. Values with leading zeroes are allowed. Let's look at the following possible matrix with 4 rows and 7 columns: 
1849312
8951400
5678129
8721365
The path going down, right, right, down, right, right, right, down, right has value 1895781265. Other possible paths have values 1849312095, 1895781365, or 1858721365, among many others.

The girl can change some (possibly all or none) of the cells of the matrix. If the current digit in a cell is X, in one operation the girl can change it to either X-1 or X+1. The digits change cyclically between 0 and 9 – thus changing 0 to 9 or 9 to 0 takes a single operation. Elly can do as many operations as she wants – thus she can get to any other matrix of digits with the same dimensions.

Elly wants to get a matrix on which all paths have values divisible by three. Of course, she wants to do it with as few operations as possible. Given the ints N and M, as well as the String[] matrix giving the initial digits in the matrix, return the minimum number of operations required to do so.

Limits

Constraints

  • N and M will be between 1 and 50, inclusive.
  • matrix will contain exactly N elements.
  • Each element of matrix will contain exactly M characters.
  • Each character in matrix will be a digit ('0'-'9').
